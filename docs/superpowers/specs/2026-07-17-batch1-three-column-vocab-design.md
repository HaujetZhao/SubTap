# 字幕音频播放器 批次一设计文档（三栏布局 + 词库内置 + 分级勾选）

> 日期：2026-07-17
> 状态：待审阅
> 基线：`docs/superpowers/specs/2026-07-17-srt-audio-player-design.md`（第一版）

## 1. 背景

第一版（已交付，分支 `feature/srt-audio-player`）实现了：上传 SRT + 音频 + 词库 → 点句子播放对应片段 + 右栏显示命中单词。

用户实测后发现核心价值在于**主动点读**（先读句再听），而非被动听力。本次迭代（批次一）聚焦**布局重构 + 词库内置 + 分级控制**，为后续托管到 GitHub Pages 做准备。批次二（媒体支持 + 字幕微调）另起设计。

## 2. 目标（批次一范围）

1. **三栏布局**：左操作栏 / 中字幕栏 / 右单词栏。撤掉原顶部按钮栏。
2. **词库内置**：构建时把 `vocabulary.json` 打进 `dist/index.html`，用户无需再选词库文件。
3. **分级勾选**：左栏设置面板里可勾选启用哪些分级（默认全选）。
4. **右栏按级分栏**：命中单词按分级聚集显示，每级一个标题 + 单词列表；不显示分级小标签了；未命中或未勾选的分级不显示该栏。

## 3. 非目标（YAGNI，本批不做）

- 视频支持（批次二）。
- 字幕时间戳微调（批次二）。
- 文件管理功能（左栏预留，本批只放设置）。
- 词库云端更新、用户自定义词库（内置即固定，更新靠重新构建）。
- 词库分级的新增/编辑（只读勾选）。

## 4. 架构变更

### 模块职责（变更/新增部分）

| 模块 | 变更 | 职责 |
|------|------|------|
| vocab-store.js | **新增** | 持有内置词库（构建期注入）、分级列表、勾选状态；提供 `getEnabledLevels()`、`setEnabled(level,bool)`、`lookupByLevel(text)` 返回 `{level: Word[]}` 分组结果 |
| ui.js | **重写** | 渲染三栏；设置面板（分级勾选）；右栏按级分栏；句子列表/播放态沿用第一版逻辑 |
| main.js | **调整** | 移除 `vocab-input` 处理；接入 vocab-store；设置变更触发右栏刷新 |
| build.js | **调整** | 构建时把 vocabulary.json 内容注入一个 `<script>` 内联变量 |
| styles.css | **大改** | 三栏 grid/flex 布局 + 设置面板 + 分栏样式 |
| index.html | **重写 body** | 三栏结构；移除顶部按钮栏和词库上传项 |

不变模块：srt-parser.js、word-lookup.js、player.js（其 `App.buildVocab` 仍被 vocab-store 内部复用）。

### 数据流

1. 页面加载 → vocab-store 用内置词库初始化（`buildVocab` → 大表 + 保留分级信息）。
2. 用户上传 SRT + 音频 → 句子列表渲染到中栏。
3. 用户点句子 → 高亮 + 播放片段 + 右栏刷新。
4. 右栏刷新：vocab-store 按当前勾选的分级，对句子查词并**按级分组** → ui 渲染分栏。
5. 用户在设置面板勾/取消某分级 → vocab-store 更新状态 → 若当前有选中句，右栏立即按新勾选重新刷新。

## 5. 关键实现要点

### 5.1 词库内置（build.js 注入）

构建期读取 `vocabulary.json`，序列化为 JSON 字符串，注入到 dist 的一段内联脚本：

```html
<script>window.__VOCAB__ = { ... }; </script>
```

开发期（`index.html` 直接打开）没有这段注入，故 vocab-store 需兼容：若 `window.__VOCAB__` 不存在，则**回退到 fetch 本地 `vocabulary.json`**（开发体验）；存在则直接用。这样开发页和 dist 都能跑。

> 注：dist 里 fetch 本地文件会被 `file://` 协议限制，所以**内置注入是 dist 的唯一可靠来源**；开发页靠 fetch 兜底。

### 5.2 分级勾选状态（vocab-store）

- 分级顺序固定：`初中 / 高中 / 四级 / 六级 / 考研 / 托福 / SAT`（取自词库 key 顺序）。
- 默认全选。
- 勾选状态保存在 vocab-store 内（本批不持久化到 localStorage——刷新即重置，符合"刷新即丢"的第一版约定；持久化留待后续）。

### 5.3 按级查询（vocab-store.lookupByLevel）

复用 `App.lookupWords` 得到扁平 `Word[]`，再按 `word.level` 分组成 `{level: Word[]}`，**只保留已勾选的分级**，且**保留句中出现的顺序**（每级内按单词在句中首次出现排序）。

### 5.4 右栏分栏渲染（ui）

- 遍历分级顺序，对该级有命中单词的，渲染一个分栏：标题（分级名 + 该级词数）+ 单词列表（单词 + 释义，无小标签）。
- 无命中的分级不渲染该栏。
- 句中无任何命中词时显示占位"当前句没有词库中的单词"。

### 5.5 三栏布局（styles）

- 整页 `display:flex`，横向三栏。
- 左栏：固定宽度（如 220px），顶部标题，下方设置面板（分级勾选）。
- 中栏：`flex:1`，上方预留视频区占位（本批留空 div，批次二填），下方句子列表可滚动。
- 右栏：固定宽度（如 300px），单词分栏，可滚动。
- 响应式：窄屏（<900px）本批不做专门适配，保证桌面可用即可。

## 7. 错误处理

| 情况 | 处理 |
|------|------|
| 内置词库缺失（开发页且 fetch 失败） | 左栏设置区显示"词库加载失败"，查词功能降级（右栏始终占位） |
| SRT/音频解析失败 | 沿用第一版：状态条红字提示，不崩 |
| 全部分级被取消勾选 | 右栏占位"未勾选任何分级" |

## 8. 测试

- **纯函数**（vocab-store 的按级分组逻辑）：扩展 `test.html`，新增断言：给定扁平 Word[] 和勾选集合，分组结果正确、顺序正确、未勾选级被过滤。
- **浏览器手动验收**：见 §9。
- buildVocab/lookupWords/parseSRT 沿用第一版已有断言，不重测。

## 9. 验收标准（手动，用真实文件）

1. 打开 `dist/index.html`（无需选词库——已内置）。
2. 页面呈三栏；左栏有 7 个分级勾选项（默认全选）。
3. 选 SRT + 音频 → 中栏出句子列表。
4. 点句子 → 播放 + 高亮 + 右栏按级分栏显示命中词（无分级小标签，分级名作栏标题）。
5. 在左栏取消勾选"四级" → 右栏立即不再显示四级那栏（若当前句有四级词）。
6. 取消所有勾选 → 右栏显示"未勾选任何分级"。
7. 开发页 `index.html`（未内置词库）能通过 fetch 兜底加载词库，功能一致。
8. dist/index.html 单文件可独立拷走使用（不依赖 vocabulary.json）。

- 构建：`node build.js` → 注入词库 + 内联 css/js → `dist/index.html`。
- 日常/托管：用 `dist/index.html`，单文件可托管到 GitHub Pages。

## 12. 与批次二的衔接

本批在中栏顶部预留"视频区占位"div；批次二填充视频元素与控制。左栏"文件"区本批只放字幕+音频入口，批次二加视频入口和字幕微调参数。三栏布局结构本批即定型，批次二不动骨架。
