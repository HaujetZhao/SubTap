# 字幕音频句段播放器 设计文档

> 日期：2026-07-17
> 状态：待审阅

## 1. 目标

做一个**单文件 HTML 网页**，用户在浏览器里打开后：

1. 上传 SRT 字幕文件、音频文件、词库 JSON（`vocabulary.json`）。
2. 网页自动解析 SRT，把每条字幕渲染成"句子列表"。
3. 点击任意句子 → 直接播放音频中对应的那个时间段（无完整时间轴，体验像独立片段）。
4. 右侧"单词面板"自动列出当前句中能在词库里查到的单词及其释义。

## 2. 非目标（YAGNI）

- 不做服务端、不做数据库、不做云存储。纯本地浏览器运行。
- 不做音轨剪辑/导出音频片段。
- 不做用户账号、笔记持久化（刷新即丢，本期可接受）。
- 不做译文翻译（字幕本身无译文，词库释义已够）。
- 不做波形图、频谱可视化。

## 3. 输入格式约定（基于实际文件确认）

### SRT（标准格式）
```
1
00:00:00,080 --> 00:00:01,480
When you encode text into binary

2
00:00:02,080 --> 00:00:04,200
it's often nice to use as little data as possible
```
- 序号行 → 时间轴行（`HH:MM:SS,mmm --> HH:MM:SS,mmm`）→ 一行或多行文本 → 空行分隔。
- 时间精度到毫秒。

### 音频
- 优先支持 `.mp3`（浏览器原生支持，已确认 `test.mp3` 存在）。
- `.m4a` 多数现代浏览器也支持（AAC），但作为后备；若加载失败，提示用户改用 mp3。
- 通过 `URL.createObjectURL(file)` 生成临时 URL 喂给 `<audio>`。

### 词库 `vocabulary.json`（两级结构）
```json
{
  "初中": {"boat": "n. 船，小船", "group": "...", ...},
  "高中": {...},
  "四级": {...},
  ...   // 共 7 个分级
}
```
- 外层 key = 分级名（初中/高中/四级…），内层 = `单词 → 释义`。
- **查询前需把所有分级合并成一张 `{ 单词: {level, def} }` 大表**，查到时连带显示分级。

## 4. 架构

单文件 `index.html`，内含 HTML + CSS + JS（无外部依赖、无构建步骤）。

JS 按职责拆成 **6 个模块（闭包/对象）**，每个职责单一、可独立测试：

```
┌──────────────────────────────────────────────┐
│                    UI 层（DOM 事件绑定）        │
└──────┬───────────────────────────────────────┘
       │ 调用
┌──────▼──────────┐   ┌─────────────────┐
│ 1. FileLoader    │   │ 4. Player        │
│  读取3类文件      │   │  隐藏audio+播放   │
└──────┬──────────┘   │  指定区间          │
       │              └────────▲─────────┘
┌──────▼──────────┐            │ playSegment
│ 2. SrtParser     │            │
│  SRT文本→句子[]  │   ┌────────┴─────────┐
└──────┬──────────┘   │ 3. SentenceList   │
       │ sentences    │  渲染/高亮/点击     │
┌──────▼──────────┐   └────────┬──────────┘
│ 6. PanelRenderer │ <─────────┤ 当前句
│  右侧单词面板     │   ┌───────▼──────────┐
└──────▲──────────┘   │ 5. WordLookup      │
       │ words         │  句子→分词→查词库   │
       └───────────────┘                    │
```

### 模块职责

| # | 模块 | 输入 | 输出 | 说明 |
|---|------|------|------|------|
| 1 | FileLoader | `<input>` change 事件 | `{audioUrl, srtText, vocab}` | 读文件、解析 JSON、为音频建 objectURL |
| 2 | SrtParser | SRT 文本（字符串） | `Sentence[]`：`{id,start,end,text}` | **纯函数**，可单测 |
| 3 | SentenceList | `Sentence[]` | 渲染 DOM 列表；暴露"点句子"回调 | 每句显示 `[mm:ss] 文本` |
| 4 | Player | `{audioUrl}` | `playSegment(start,end)`、`stop()` | 隐藏 `<audio>`，`timeupdate` 到 end 自动停 |
| 5 | WordLookup | `(sentenceText, vocab大表)` | `Word[]`：`{word, level, def}` | **纯函数**：小写化、分词、去标点、查表 |
| 6 | PanelRenderer | `Word[]` | 渲染右侧面板 | 按出现顺序、去重 |

### 数据流

1. 用户依次选 SRT / 音频 / 词库 → FileLoader 读出三者。
2. SrtParser 把 SRT 变成 `Sentence[]` → SentenceList 渲染左侧。
3. 用户点某句 → SentenceList 触发回调 → 同时做两件事：
   - 调 `Player.playSegment(start, end)`；
   - 调 `WordLookup(sentence.text)` → PanelRenderer 刷新右面板 + 高亮当前句。

## 5. 关键实现要点

### 5.1 播放指定区间（复用已验证的示例逻辑）
- 一个隐藏的 `<audio id="audio" preload="metadata">`。
- `playSegment(start,end)`：设 `currentTime=start` → `play()` → 监听 `timeupdate`，到 `end` 则 `pause()`。
- 切句时先清掉旧的 `timeupdate` handler，避免串扰。

### 5.2 SRT 解析（容错）
- 按空行分块；每块首行若是纯数字（序号）则跳过。
- 用正则 `(\d{2}):(\d{2}):(\d{2})[,.](\d{3})\s*-->\s*(\d{2}):(\d{2}):(\d{2})[,.](\d{3})` 抓时间轴（同时兼容 `,` 和 `.` 毫秒分隔）。
- 文本部分可能多行，用 `\n` 连接后作为一句。
- 解析失败时给出"第 N 块格式异常"的提示，不整体崩溃。

### 5.3 单词查询
- 词库加载后**预构建大表**：遍历 7 个分级，`word → {level, def}`，存进一个普通对象。
- 句子分词：转小写 → 用 `/[^a-z']+ /` 拆分 → 去标点 → 过滤空串。
- 查大表：能查到的进面板，查不到的**不显示**（避免噪声）。
- 同一单词在句中重复只显示一次。

### 5.4 时间戳显示
- 句子前缀 `[mm:ss]` 用 `start` 秒数格式化（不足两位补零）。

## 6. UI / 布局

```
┌────────────────────────────────────────────────────┐
│ 选字幕 .srt ▢  选音频 ▢  选词库 .json ▢    [状态条]  │  顶部上传区（固定）
├──────────────────────────────────────┬─────────────┤
│                                      │  当前句单词   │
│  句子列表（可滚动主体）                 │             │
│  ┌────────────────────────────────┐  │  ▸ 四级       │
│  │ [00:00] ▶ When you encode...   │  │    compress  │
│  │ [00:02]   it's often nice...   │  │    v. 压缩    │
│  │ [00:04] ▶ So you might...      │  │             │
│  └────────────────────────────────┘  │  ▸ 初中      │
│  （点任意行 → 播该段 + 高亮 + 刷右栏）   │    encode    │
│  当前播放行：浅蓝底；正在播：左侧▶红     │    v. 编码    │
│                                      │             │
└──────────────────────────────────────┴─────────────┘
```
- 当前句：浅蓝背景；正在播放时：左侧图标变红 `⏸`，其余是 `▶`。
- 句子点击区域整行可点（不只是图标）。
- 词库未加载时右栏显示"请上传词库"；句子未选时显示"点击左侧句子"。
- 简洁现代风：系统字体、圆角、柔和阴影，无花哨配色。

## 7. 错误处理

| 情况 | 处理 |
|------|------|
| SRT 解析失败 | 顶部状态条红字提示"字幕第 N 段格式异常"，已解析成功的句子仍可用 |
| 词库 JSON 解析失败 | 状态条提示"词库格式错误"，播放功能不受影响（右栏显示提示） |
| 音频无法播放（如编码不支持） | `<audio>` 的 `error` 事件 → 状态条提示"音频无法播放，建议改用 mp3" |
| 三者未全部上传 | 句子列表/播放按已上传的部分工作；缺音频则点句子提示"请先选音频" |

## 8. 测试

**纯函数模块用浏览器控制台或简单断言点测：**

- **SrtParser**：
  - 标准单句 → 正确 `{start,end,text}`。
  - 多行文本 → 合并为一句。
  - 序号行被正确跳过。
  - 毫秒分隔符 `,` 和 `.` 都能解析。
  - 拿真实的 `【官方双语】…熵.srt` 跑一遍，句数与肉眼数一致。
- **WordLookup**：
  - 句中单词命中词库 → 返回带 level 和 def。
  - 同词重复 → 去重。
  - 词库未含的词 → 不返回。
  - 大小写、标点 → 正确归一化。
  - 拿真实 `vocabulary.json` + 真实句跑一遍。

播放器、UI 用**手动验收**（见下）。

## 9. 验收标准（手动）

用项目里的真实文件测试：

1. 打开 `index.html`，依次选 `.srt`、`.mp3`、`vocabulary.json`。
2. 左侧出现句子列表，每条带 `[mm:ss]` 前缀。
3. 点第一句 → 从第 0 秒开始播，到该句结束自动停。
4. 点中间某句 → 跳到对应时间播，到结束自动停；当前句高亮。
5. 右侧单词面板显示该句中能查到的单词 + 分级 + 释义。
6. 切换句子时，上一句停止、当前句高亮、右栏刷新。
7. 选 m4a 也能播（若浏览器支持）；不支持时给出明确提示。

- 测试时用现有文件：`【官方双语】压缩即智能：Part1，重新发明熵.{srt,mp3}` + `vocabulary.json`。
